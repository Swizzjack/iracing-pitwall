## 1. Architecture Overview

```
┌──────────────────┐    MMF (60Hz)      ┌─────────────────┐    WebSocket JSON   ┌──────────────────┐
│ iRacing.exe      │ ─────────────────► │ Rust Bridge     │ ──────────────────► │ React Dashboard  │
│ (Win, 64-bit)    │  IRSDKMemMapFile   │ (single .exe)   │   ws://127.0.0.1    │ (embedded HTML)  │
│ + YAML Session   │  IRSDKDataValid    │                 │                     │                  │
└──────────────────┘    Event           └─────────────────┘                     └──────────────────┘
                                              │
                                              ▼
                                        rust-embed serves
                                        compiled dashboard/dist
                                        on same port as WS
```

**Stack:**
- Bridge: Rust 2021, tokio, tokio-tungstenite, serde + serde_yaml, encoding_rs (ISO-8859-1), ts-rs v10, thiserror, anyhow, log, env_logger, windows-sys 0.59.
- Cross-compile: `CARGO_HOME=/work/.cache/cargo cargo zigbuild --release --target x86_64-pc-windows-gnu --manifest-path bridge/Cargo.toml` (CARGO_HOME nötig, da /opt/cargo read-only).
- Frontend: React 18 + TypeScript + Vite. Tailwind. Brand: `#0f0f0f` / `#facc15` / `#f97316`, Teko + Roboto Condensed + JetBrains Mono.
- Shared types: `shared/` directory, generated by ts-rs (`cargo test --manifest-path bridge/Cargo.toml`).
- Distribution: Single `.exe` via `rust-embed`.

## 4. iRacing SDK Reference (cheat sheet)

**Names (use raw strings, exact case):**
- MMF: `r"Local\IRSDKMemMapFileName"`
- Event: `r"Local\IRSDKDataValidEvent"` — auto-reset, signaled at 60Hz when new frame ready

**Header (at MMF offset 0):**
```rust
#[repr(C)]
struct Header {
    version: i32,            // expect 2
    status: i32,             // 1 = connected
    tick_rate: i32,          // 60
    session_info_update: i32,
    session_info_len: i32,
    session_info_offset: i32,
    num_vars: i32,
    var_header_offset: i32,  // -> array of 336 VarHeader (144 bytes each)
    num_buf: i32,            // 3 (triple buffer)
    buf_len: i32,            // 7831 bytes per buffer
    pad1: [i32; 2],
    var_buf: [VarBuf; 4],    // only num_buf entries valid
}

#[repr(C)]
struct VarBuf {
    tick_count: i32,
    buf_offset: i32,         // offset into MMF where this buffer starts
    pad: [i32; 2],
}
```

**VarHeader (144 bytes, packed by SDK convention but `repr(C)` works since fields naturally align):**
```rust
#[repr(C)]
struct VarHeaderRaw {
    var_type: i32,           // 0=char/bool, 1=int, 2=bitfield(u32), 3=f32, 4=f64
    offset: i32,             // into the 7831-byte frame buffer
    count: i32,              // 1 for scalars, N for CarIdx*
    count_as_time: u8,       // bool
    _pad: [u8; 3],
    name: [u8; 32],          // null-terminated
    desc: [u8; 64],
    unit: [u8; 32],
}
```

After parsing, store as:
```rust
pub struct VarDescriptor {
    pub var_type: VarType,    // enum
    pub offset: usize,
    pub count: usize,
    pub unit: String,
}

pub enum VarType { Char, Int, Bitfield, Float, Double }
```

**Triple-buffer read protocol:**
1. `WaitForSingleObject(event, INFINITE)`
2. Scan `header.var_buf[0..num_buf]`, pick `i` with max `tick_count`. Save `tick_a`.
3. Copy `buf_len` bytes from `mmf[var_buf[i].buf_offset..]` into local `Vec<u8>`.
4. Re-read `var_buf[i].tick_count` → `tick_b`. If `tick_b != tick_a`, retry from step 2 (max 5 attempts).
5. Frame is now stable in local buffer.

**windows-sys 0.59 imports for Task 3+4:**
- `Win32::System::Threading::WaitForSingleObject`
- `Win32::System::Threading::SYNCHRONIZATION_SYNCHRONIZE` *(NOT `SYNCHRONIZE`)*
- `Win32::System::Memory::*` (already used in Task 1)
- `Win32::Foundation::{HANDLE, CloseHandle, INFINITE, WAIT_OBJECT_0}`

**YAML SessionInfo:**
- Bytes at `header.session_info_offset` for `header.session_info_len` bytes.
- Encoding: ISO-8859-1 → use `encoding_rs::WINDOWS_1252.decode(bytes).0`.
- Re-parse only when `header.session_info_update` changes.

**Files:** `bridge/src/iracing_sdk/reader.rs`, `bridge/src/main.rs`.

**API to add:**
```rust
impl IRacingClient {
    /// Blocks until iRacing signals a new frame, then snapshots it via
    /// triple-buffer protocol. Stores result in `self.frame: Vec<u8>`.
    pub fn wait_for_frame(&mut self) -> Result<()>;

    pub fn get_f32(&self, name: &str) -> Result<f32>;
    pub fn get_f64(&self, name: &str) -> Result<f64>;
    pub fn get_i32(&self, name: &str) -> Result<i32>;
    pub fn get_bool(&self, name: &str) -> Result<bool>;          // var_type Char
    pub fn get_bitfield(&self, name: &str) -> Result<u32>;
    pub fn get_f32_array(&self, name: &str) -> Result<&[f32]>;    // CarIdx*
    pub fn get_i32_array(&self, name: &str) -> Result<&[i32]>;
    pub fn get_bool_array(&self, name: &str) -> Result<Vec<bool>>;
}
```

**Type-check helper (private):**
```rust
fn read<T>(&self, name: &str, expected: VarType) -> Result<&[T]>
where T: Copy;
```
Returns slice over `self.frame[offset..offset + count*size_of::<T>()]` after
verifying `var_type == expected`.

```rust
        client.get_f32("Speed")?,

        client.get_f32("RPM")?,

```

### Task 8 — WebSocket server

**Files:** `bridge/src/ws/handler.rs` (serve() implementieren), `bridge/src/ws/protocol.rs`
(ServerMessage-Enum bereits da).

Tatsächliches Enum (in `ws/protocol.rs`):
```rust
pub enum ServerMessage {
    Hello { bridge_version: String },
    SdkStatus { status: HeaderStatus },
    Telemetry { snapshot: TelemetrySnapshot },
    Standings { snapshot: StandingsSnapshot },
    SessionInfo { info: SessionInfoYaml },
    // …
}
```

**Topology (in `main.rs`):**
- Task A — telemetry loop (`spawn_blocking`): `wait_for_frame()` → `TelemetrySnapshot::build()` → broadcast.
- Task B — standings loop (250ms interval): YAML re-parse bei `session_info_update`-Änderung → `StandingsSnapshot::build()` → broadcast.
- Task C — WebSocket listener auf `127.0.0.1:8765` (Env: `BRIDGE_WS_PORT`):
  - Accept connections; per-connection task subscribes to both broadcasts and forwards as `Message::Text(serde_json::to_string(&msg))`.
  - On client disconnect, drop subscription. No backpressure: lagged receivers re-subscribe automatically with a `Connection{true}` resync.

**Port:** `8765` (tatsächlicher Default, nicht 9013). Env: `BRIDGE_WS_PORT`.
